# Perspective Rotation

Rotate through reviewer/practitioner/theorist/time-machine/novice perspectives systematically.

## Stages

### Stage 1: Reviewer 2 Hat

Adopt hostile reviewer stance. Find fatal flaws, logical gaps, missing evidence, and unstated assumptions. Use reviewer2-hat SOP.

### Stage 2: Practitioner Hat

Switch to engineer perspective. Assess buildability, cost, timeline, technical debt, and integration challenges. Use practitioner-hat SOP.

### Stage 3: Theorist Hat

Adopt theorist role. Assess formal foundations, mathematical rigor, theoretical completeness, and formalization opportunities. Use theorist-hat SOP.

### Stage 4: Time Machine

Project to multiple time horizons. How does this look from 5yr/50yr/500yr future? What's obsolete, what endures? Use time-machine SOP.

### Stage 5: Novice Perspective

Become deliberate novice. Question every "obvious" assumption. Ask "why?" repeatedly. Find hidden complexity masquerading as simplicity. Use novice-perspective SOP.

### Stage 6: Synthesize

Combine all perspective outputs into integrated multi-perspective assessment. Identify convergent insights and divergent tensions. Use perspective-synthesis SOP.

## Minimum Yield

| Metric | Floor |
|--------|-------|
| Perspectives applied | ≥5 |
| Unique insights per perspective | ≥3 |
| Cross-perspective convergences | ≥2 |
| Actionable improvements identified | ≥5 |
