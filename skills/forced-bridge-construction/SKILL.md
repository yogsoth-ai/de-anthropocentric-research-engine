# Forced Bridge Construction

Force connections between unrelated technologies. Unlike natural analogy discovery, this strategy deliberately constructs bridges where none exist, using the construction process itself as a creative mechanism.

|-----|------|
| forced-connection | Core SOP — force connection between concept pairs |
| domain-scanning | Identify candidate technologies for bridging |

| structural-mapping | Map structural correspondences across the bridge |
| transfer-adaptation | Adapt bridged concept to target constraints |
| cross-domain-synthesis | Synthesize all bridge-derived solutions |

## Execution Guidance

1. **Select endpoints**: Choose two unrelated technologies/domains via domain-divergence
2. **Identify abstraction levels**: For each endpoint, extract principles at multiple abstraction levels
3. **Construct bridge**: Use forced-connection SOP to build a path between endpoints
4. **Test bridge load**: Can the bridge carry a real solution concept? Use bridge-validation
5. **Iterate endpoints**: Try ≥4 different endpoint pairs
6. **Extract solutions**: From validated bridges, derive concrete solution ideas
7. **Synthesize**: Combine bridge-derived ideas into coherent proposals
