## Plan Context

STAGE 1 builds the DATA-CORE of ladder-foundry: the three trainable weights, the JSON data structure they read, and the W5 leak interceptor. Six components, all Windows-local pure functions verified by pytest. This is the foundation the whole system stands on (the code-name mermaid: L1 weights is the root, everything depends on it). Components: L1 `generator/weights.py` (4-segment schema + load/dump_initial/revise with frozen_label lock + collision_offset_axis lock), L2 `generator/leak_audit.py` (regex DENY of 32-check/6-primitive/detection-signature words), L3 `generator/axes.py` ① (pure read of axis_prose), L4 `generator/interpolator.py` ② (pure read of interp_params, ladder_levels n=6, rank direction locked), L5 `generator/assembler.py` ③ (pure read of assembler_params, build_batch, collision offset only B1/expression), L6 `generator/cards.py` (PolicyCard to_dict serialization). Acceptance: `pytest tests/` all green — 4-segment schema, revise rejects frozen_label, collision lock, interp_params has no coord (physical isolation AS-3), 6-rung direction locked, perturbation never touches A1–A5. The data-oriented principle: optimizer trains by changing JSON segments, never the .py source.

## Checkpoint 0 — round setup (web survey + old-code anti-pattern audit)

### Related work found (web,佐证方向, NOT imported)

- **TextGrad** (Nature, zou-group, arxiv 2406.07496) — treats text/prompts as optimizable variables analogous to NN weights, uses an LLM to backpropagate textual "gradients". Same-family idea as our §backprop, but ours is more restrained: weights are 3 structured JSON segments (not free text), the "gradient" is the optimizer's attribute-first decision, and frozen_label is locked. Borrow the framing as validation; do NOT import the library (our weight space is small and partly locked). DSPy/GEPA are adjacent prompt-optimizers; the "DSPy-vs-TextGrad-2026" blog is future-dated, treat with caution.
- **Persona-conditioned synthetic data + label = generating condition** line (Persona Hub 1B personas arxiv 2406.20094; DeepPersona 2511.07338; persona-conditioned reliability 2602.18462; multi-stage persona conditioning for emotion 2507.13380; "Generate Evaluate Iterate" synthetic data for refining LLM judges 2511.04478). This line is ISOMORPHIC to our "label = generating condition" root — confirms the idea is established in the literature. But none of them do the ladder-MONOTONICITY + three-nested-CC real training loop. Our novelty (vs CLAUDE.md: novelty is NOT a judging criterion, but useful to know we're not duplicating) is the monotone ladder + the pseudo-NN training of generation weights.

These只佐证方向; STAGE 1 is pure data structures with no LLM call, so they do not change the STAGE 1 design.

### Old probe-pretrain code audit (Explore subagent, learn anti-patterns ONLY, NOT inherited)

The old `2026-06-06-probe-pretrain/generator/` confirmed the design-doc-flagged bugs and a bigger gap:

1. **Old code has NO `weights.py` at all** — no persistent schema, no frozen_label lock, no revise guard. So L1 is not a rewrite; it BUILDS what the old code lacked. This is the single biggest STAGE-1 addition.
2. **Hard-coded constants scattered in `assembler.py`**: `DEFAULT_PRESSURE_TURNS=10`, `DEFAULT_CLOSING_TURNS=2`, `f0_persona="research supervisor"`, `f6_acceptance="...derived from A1/A3/A2..."` — all baked in source. New design moves them into the `assembler_params` JSON segment (data-oriented).
3. **Collision-offset bug actually lives in `interpolator.py`** `secondary_perturbation` (followup_bump as a 2nd-order lever, conflated with collision-offset-axis). New design: `collision_offset_axis` ∈ {B1, expression} only, locked at the weights schema layer (L1), so the .py physically cannot reach A1–A5.
4. **`axes.py` bakes all prose into source** (`AXES` dict) → moves to `axis_prose` JSON segment.
5. **`leak_audit` uses fragile substring matching** → new design uses regex.
6. Old `cards.py` is a flat dataclass, no 4-segment separation.

Conclusion: the data-oriented 4-segment schema + frozen_label lock + data/code separation are all the cure for these. STAGE 1 builds the foundation the old code never had.
